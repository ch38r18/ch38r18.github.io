开始hugo的博客之旅

用md编辑器整理自己的所有东西，当想发布的时候，就把要分享的文章分享出去

第一个问题时区

新建文件的固定格式 archetypes\default.md

title: "{{ replace .Name "-" " " | title }}"
date: {{ .Date }}
draft: true

修改：draft草稿、date时区问题

title: "{{ replace .Name "-" " " | title }}"
date: {{.Date}}+08:00

PC端的date 有时区，手机termux端没有时区，怎么办

第二个问题局域网查看

局域网查看命令和hexo s不一样，不能只用hugo 还得又其他参数。

hugo server -b "http://192.168.199.216" --bind "192.168.199.216"
hugo server -b "http://192.168.199.218" --bind "192.168.199.218"
hugo server -b "http://192.168.199.216" -p 8080 --bind "192.168.199.216"

第三个问题上传hugo到github仓库

hugo生成静态网页文件

hugo

在 public中就是生成好的静态网页文件，可以用浏览器打开

把public重命名未我的博客 ch38r18.github.io 和github上的仓库名称一致

在上面文件夹中，git bash打开，执行一下代码

git init  //初始化git
git add .
git commit -m "第一次上传"  //提交更新，备注信息
git push -u origin master
//验证用户

让后到我的https://ch38r18.github.io/ 中查看效果

第四个问题更新hugo

当文件夹结构更换以后

生成静态网页 public 文件夹

hugo --baseUrl="https://ch38r18.github.io/"

将photos 移动到 my photos 中，否则相册无法正常显示

删除远程仓库博客数据v01

git pull git@github.com:ch38r18/ch38r18.github.io.git //将仓库与本地同步
git rm * -r //-r 当给出主目录名时允许递归删除，删除本地仓库中所有文件
git add .
git commit -m "删除博客数据v02"  //提交本次删除数据
git push

替换上传博客数据v02

//把生成的 public 新数据复制到目录中 执行下面参数，上传
git add .
git commit -m "2021.01.29-上传博客数据v02"  //提交本次上传数据
git push -u origin master --force  //--force强制覆盖参数
//用户验证 帐号 密码

等待12分钟，刷新 https://ch38r18.github.io/才看到效果
